Section : SMBIOS Check (10) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
1. use the sm_bios tool to get the smbios info logs .

2. compare the txt log with BIOS mini SPEC ->'System Management BIOS' chapters
BIOS Information_Type 0
[Offset 04h] Vendor
Must be "LENOVO"
[Offset 05h] BIOS Version
xxCNmmWW, xx:BIOS ID number;mm:BIOS release number <MUST BE DISPLAYED IN BIOS SETUP>
[Offset 8h] BIOS Release Date
String number of BIOS release date

[Offset 0Ah] BIOS Characteristics,System Vendor



Note： From Gen X SMB product, check information with smbBC.exe.


Run CMD as administrator：smbBC.exe

For Consumer Product:


Bit48:50 : 3bit value = Enclosure type


These three bits must be set to: 02h = Portable


Bit51:55 : 5bit value = Brand type


03h = Lenovo Consumer Notebook


Bit56:63 : reserved all other are reserved


 


For SMB Product:


Brand Type: 0x01h(Lenovo)


Enclosure Type Extension: 0x01(Lenovo SMB Product)


 


The other data in this field follows the SMBIOS standard definitions.
[Offset 14h] System BIOS Major Release
Always 0 prior to shipping the product; Always 1 after a product ships
[Offset 15h] System BIOS Minor Release
Same as "mm" in BIOS Version
[Offset 16h] EC Major Release
Always 0 prior to shipping the product; Always 1 after a product ships
[Offset 17h] EC Minor Release
Same as "mm" in EC Version
System Information_Type 1
[Offset 04h] Manufacturer
Must be "LENOVO"
[Offset 05h] Product Name
MT (Machine Type) assigned by Lenovo. Default = “INVALID”, Lenovo require to support MT string length up to 64 bytes.
[Offset 06h] Version
ASCIIZ string for product name, for example: ”LenovoTianYi Fxx”, system must support product name length up to 32 bytes.
[Offset 07h] Serial Number
Lenovo SN. Default =“INVALID”, Actual Lenovo SN must be injected during manufacture process.system must support Lenovo SN length up to 32 bytes.
[Offset 08h] UUID
UUID, 16 bytes.Default:"INVALID" <MUST BE DISPLAYED IN BIOS SETUP>

[Offset 19h] SKU Number

BIOS should fill SKU number depending on MTM(Machine Type and Model number).


If MTM exist, SKU number string is “LENOVO_MT_XXXXX_BU_idea_FM_YYYY”, else SKU number string is “LENOVO_BI_IDEAPADXX_BU_idea_FM_YYYY”.


for example: LENOVO_BI_IDEAPAD66_BU_idea_FM_Lenovo Yoga 13.


Please reserve 64 bytes for the SKU string.


String “LENOVO_MT_X XXXX” The XXXXX is Machine Type, Machine Type is five or four letters or numbers.


String “LENOVO_BI_IDEAPADXX”. The XX is BIOS build ID with two letters or numbers.


YYYY is family string in SMBIOS type 1 offset 1Ah

[Offset 1Ah] Family

Product name string, please confirm with your technical project manager


Reserve 64 bytes. Default = “IDEAPAD”


The Family field should be populated with the full marketing product name, same as the model name in Rating Label.


If ‘Lenovo’ is part of the product name and it shall NOT be repeated in the ‘Family’ SMBIOS field.


If product name contains multi-byte characters (CHS, JPN, etc.), please check with Lenovo TPM for its English alias name.


The value must be consistent after mass-production, no changes to value or format.
Base Board Information_Type2
[Offset 04h] Manufacturer
Must be "LENOVO"
[Offset 05h] Product
Project name string, system must support project name string length up to 32 bytes.
[Offset 06h] Version
OS license information. OS license information = OS PN + OS description. Default= No DPK, the OS license information must be displayed in BIOS setup and a space must be added between OS PN and OS description in setup menu
[Offset 07h] Serial Number
Lenovo SN. Default =“INVALID”,Actual Lenovo SN must be injected during manufacture process.system must support Lenovo SN length up to 32 bytes. <MUST BE DISPLAYED IN BIOS SETUP>
[Offset 08h] Asset Tag
ASCIIZ string, default is “No Asset Tag”. When setup Asset Tag number, BIOS must build up this field with the Asset Number.
System Enclosure or Chassis_Type 3
[Offset 04h] Manufacture
Must be "LENOVO"

[Offset 05h] Type

Follow project define, type value should be one of the four types for Ideapad.


1. Notebook: System with an integrated display in a clamshell form factor.


2. Convertible: System that combines a display, attached keyboard that can be flipped or twisted out of the way so as to allow system to be used as a tablet, and a battery. This would include Lenovo “Yoga” devices.


3. Tablet: System that uses it’s display as the primary method of input, does not physically attach to a keyboard and includes a battery as the primary power source.


4. Detachable: System that combines a display, keyboard that can be physically attached or detached by the customer (including both systems where keyboard is sold with the system or sold separately) and a battery. This would include Lenovo systems such as the Miix and X1 Tablet form factors.


Note: please confirm with technical project manager and make sure the value is correct for your project.
[Offset 06h] Version
ASCIIZ string for product name,for example:"Lenovo IdeaPad Y410"

[Offset 07h] Serial Number

Lenovo SN.


Default =“INVALID”,Actual Lenovo SN must be injected during manufacture process.


system must support Lenovo SN length up to 32 bytes. <MUST BE DISPLAYED IN BIOS SETUP>
[Offset 08h] Asset Tag
ASCIIZ string, default is “No Asset Tag”. When setup Asset Tag number, BIOS must build up this field with the Asset Number.
Processor Information_Type 4
[Offset 10h] Processor Version
Ensure the tool show information , it is the same as your unit actual processor .
[Offset 16h] Current Speed
show CPU frequency .
[Offset 23h] Core count
Ensure the tool show core count , it is the same as your unit actual processor .

[Offset 09h] Install size

L1 cache :


for CPU support L1 cache , Ensure the tool show cache size , it is the same as your unit actual processor .


L2 cache :


for CPU support L2 cache , Ensure the tool show cache size , it is the same as your unit actual processor .


L3 cache :


for CPU support L3 cache , Ensure the tool show cache size , it is the same as your unit actual processor .


Note:  If the CPU supports P-core + E-core,  test tool read L3 cache will be twice the actual processor.

OEM Strings_Type 11 for Modern Preload and AIPC


 


Note:


1. Type 11 can be checked using the RW tool, RW tool->SMBIOS -> [OEM string] (type 11).
2. For win 11 OS, type 11 can be checked using the HE tool, HE tool->SMBIOS -> OEM string
3. Please get RW tool or HE tool from BIOS team.
4. If the RW and HE tools do not open, try the following settings:
a. Set the "VulnerableDriverBlocklistEnable" to "0" in the registry Editor (\HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\CI\Config)
b. Disable Memory integrity.(Windows security->Device security->Core isolation details->Memory integrity)
c. Restart system.

Modern preload:


If support, must show as a string "Modern Preload"


 


AIPC:


Start from Gen X projects, ff the project is defined as AIPC, must have a string show as "AIPC-MNPYY", check with platform team if not sure.


M means CPU vendor, NP means CPU ID and YY means Years of products, details can check with attachment(AIPC strings). For example: AIPC - ILL24


 


Country Code:


1 Country code string use format “Country – XXMM”, XX is ISO 3166 alpha-2 code, MM is optional. XX and MM must be injected in factory according to requirements.


2 there are spaces (0x20) before and after “-”


3 there is only one SMBIOS type 11 structure in the system


4 if XX = “MO”, “HK” and “TW”, BIOS must change the country code string to “Region – XXMM”.


5 if country code was not injected during manufacture process, BIOS must not build country code string in Type 11.
[Offset 05h] String
Show 'Modern Preload'.
Physical Memory Array _Type 16
[Offset 07h] Maximum capacity
Ensure information show system supported maximum memory capacity Please cover project's all Skus configurations.
[Offset 0Dh] Number of Memory Devices
Ensure the tool show system actual number of memory devices, including slot and onboard memory.
Memory Device_Type 17
There may be more than one Type 17 pages, one memory device should show one Type 17 page, including slot and onboard memory. Please check all Type 17 pages and cover project's all memory configuration matrix.

[Offset 0Ch] Size

Make sure memory size show correctly.



Note: If the single memory size is 32 GB-1 MB or greater, the field value is 7FFFh and the actual size is stored in the Extended Size field.
[Offset 0Eh] Form Factor
Onboard memory should show "Row of chips" Slot memory should show "SODIMM"

[Offset 04h] Hardware Security Settings

Power-on password status:


1. Enable Power on password, and ensure the status , it should show "01 enabled".


2. Disable Power on password , it should show "00 disabled".


Administrator password:


1. set the Administrator password, and ensure the status , it should show "01 enabled".


2. don’t set Administrator password , it should show "00 disabled".


Keyboard Password Status:


1. it must show "00 disabled "


Front Panel Reset Status:


1.it must show "00 disabled "
OEM Data String_Type 133
[Offset 04h] OEM Data String
it show "KHOIHGIUCCHHII" Note: If this string is found, wireless device driver is allowed to run on the system.
Product Data_Type 200
[Offser 04h] ID
Must be ASCIIZ string “IdeaPad”
[Offset 05h] MTM
Must be MTM number (machine type and model number)
[Offset 06h] BIOS SPEC Version Major number
Identifies the major release of the BIOS specification version number; for example, the value is 01h for version 1.32
[Offset 07h] BIOS SPEC Version Minor number
Identifies the minor release of the BIOS specification version number; for example, the value is 20h for version 1.32
[Offset 08h] EEPROM SPEC Version Major number
Identifies the major release of the EEPROM specification version number; for example, the value is 01h for version 1.32
[Offset 09h] EEPROM SPEC Version Minor number
Identifies the minor release of the EEPROM specification version number; for example, the value is 20h for version 1.32
[Offset 0Ah] PM SPEC Version Major number
Identifies the major release of the EM specification version number; for example, the value is 01h for version 1.32
[Offset 0Bh] PM SPEC Version Minor number
Identifies the minor release of the EM specification version number; for example, the value is 20h for version 1.32
[Offset 0Ch] SMI IO port
Chipset software SMI port address, for example, Intel chipset use B2h

[Offset 0Dh] Software SMI trigger value

the value write to SMI IO port to trigger software SMI, for IdeaPad products, this value may be 80h


Notes: Sometimes this value is depends on projects, if the value is abnormal, please confirm with project manager.
[Offset 0Eh] Lenovo variable version Major release number
Identifies the major release of the specification version number; for example, the value is 01h for version 1.32. if the system did not support Lenovo variable, this field must be FFh
[Offset 0Fh] Lenovo variable version Minor release number
Identifies the minor release of the specification version number; for example, the value is 20h for version 1.32. if the system did not support Lenovo variable, this field must be FFh
