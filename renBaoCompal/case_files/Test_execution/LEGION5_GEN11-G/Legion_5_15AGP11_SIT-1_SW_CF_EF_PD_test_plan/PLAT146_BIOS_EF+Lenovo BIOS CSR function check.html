Section : Preparation (20) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
Please confirm with project whether project support Connected System Recovery (CSR) function or not. If not support, please skip this test.

Before the CSR function test, it is necessary to first confirm with the corresponding project owner that the BIOS, preload, and drivers are ready for CSR or not.
Flash the latest BIOS and EC, cover all LAN and WLAN models of the project support

Check Software Inventory in UEFI variable. Ask the appropriate BIOS owner to provide a check tool and method, depending on the project.


Note: Software Inventory of the device embedded into a UEFI variable

The Software Inventory variable should exist. If there is no Software Inventory variable please submit an issue to TDMS

CSR test need a stable network environment.

You can capture the logs for the CSR test when failures occur and provide them to the RD team. Ignore if the test pass.



How to collect logs for Stub OS Failure:




If CSR occurs a failure with error code 0xXXXXXXXX-XXXX with a blue background， that means something error during recovery，mostly it is a network issue ， to gather logs please -

1. Format a USB sticker with FAT32
2. create a blank text file named as “cbmr_config.txt”
3. Repro the issue with sticker plugin.
4. Logs will auto gather into a folder in sticker.
5. Read the setupact .log in sysrest_ramdisk ，search errorcode for rootcause.

Section : Abnormal condition test (160) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
1. Secure Boot: Disabled
1,1 Go to BIOS setup, disable Secure Boot, check Reinstall Windows from Cloud and UEFI Wi-Fi Network Boot status.
Ensure check Reinstall Windows from Cloud and UEFI Wi-Fi Network Boot is Disabled and grayed out, cannot change setting.
2. Secure boot: Enable;Reinstall Windows from Cloud:Disabled
2.1 Go to BIOS setup,enable secure boot,disable Reinstall Windows from Cloud.

2.2 save and exit.Boot to boot menu.

Ensure Reinstall Windows from Cloud is not list.
3. Secure Boot: Enabled, Reinstall Windows from Cloud: Enabled, UEFI Wi-Fi Network Boot: Disabled
3.1 Go to BIOS setup, enable Secure Boot and Reinstall Windows from Cloud, disable UEFI Wi-Fi Network Boot.

3.2 Save and exit. Boot to Boot Menu.

For system with LAN port, Reinstall Windows from Cloud is list.


For system without LAN port, Reinstall Windows from Cloud is not list.

3.3 Select Reinstall Windows from Cloud(Only for the system have a LAN port)

Ensure system can boot to Reinstall Windows from Cloud successfully.

3.4 Select "Agree" button on prompt message windows(Only for the system have a LAN port)

Make sure the Wi-Fi connect manager is not displayed.

4. CSR abnormal UI check

4.1. Go to BIOS setup, enable Secure Boot, Reinstall Windows from Cloud, enable UEFI Wi-Fi Network Boot, save and exit

4.2 Boot to Boot Menu and Select Reinstall Windows from Cloud

Ensure system can boot to Reinstall Windows from Cloud successfully.

4.3 Select "Agree" button on prompt message windows

4.4 Connect to an unavailable network
