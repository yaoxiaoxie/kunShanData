Section : Prepare (30) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
1. Please cover all SKU for JS performance test. For detailed test scope, Please follow"JumpStart Performance test requirement“ of "lenovo SWPA requirement".
2. Refresh the latest BIOS and load BIOS to the default value.
3.Before the test, please contact BIOS owner to get the SMBIOS check tool and check the SMBIOS information.
Run the SMBIOS check tool and make sure all check items are PASS. Then begin to do the JS performance test. If not, please feedback to BIOS owner and fix it.
4.The test languages need cover EN and CS. If the project only have EN or CS ,you just need test the EN language or CS.
 
Section : JS Performance Auto Testing (60) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
*Before starting the test task，the following step must done.
1. Make sure your device is turned off, plugged in power adapter, and connect to preload server via usb dongle,if your DUT have the LAN port,you can connect to preload server directly,not use usb dongle.
2. Power on your PC ->press Fn+F12 to boot menu ->select winPE disk or EFI PXE network.
3. Make sure your network connection is successful and go to the selection list
4.Select according to the description of the latest preload step you need first seclet"SMPT",second select“PreloadGuider3-SMPT”，then select "Test"

5. Enter Smart preload test guide interface,do below select:



a. Project select preload machines.



b.OS VER select os version (e.g Win10).



c.Preload VER select preload version.



d.OS Type select os type.



e.OS Edition select os edition (For the OS edition selection: If the project shipped with both PRO and Home editions, choose PRO. If only shipped with Home edition, then select Home).



f. LanguageBox select the EN and CN test languages.
6. All Modules show all modules (App, drivers, core) selected. “>>R”,”<<L” can be used to remove/add modules.“add”can be used to add extra modules. Note: For testing Cloud OS, you need add a module "CLOUDTST" when preload.

7.1 Extra Modules show all test tools in “InActive Extra Module”:



1.[SMPT Tool].AUTOOOBE ------auto bypass all page of OOBE.



2.[SMPT Tool].AUTOBASE ------SPMT base call for main process.



3.[SMPT Tool].SMPTPELG -------collect test result and ready device for next task testing.



4.[JS Perf Testing].JSIMGPER-------- jumpstart image performance test tool(Fast startup, Standby, Edge launch and EE video playback).



5.[EE Power Switch].PowerPort 2 (JSEE02PT)



5. [EE Power Switch].PowerPort 3 (JSEE03PT)



5. [EE Power Switch].PowerPort 4 (JSEE04PT)



5. [EE Power Switch].PowerPort 5 (JSEE05PT)



5. [EE Power Switch].PowerPort 6 (JSEE06PT)



5. [EE Power Switch].PowerPort 7 (JSEE07PT)



5. [EE Power Switch].PowerPort 8 (JSEE08PT)



6. [Optional SMPT Tag].Do not run EE



6. [Optional SMPT Tag].Do not run perf

Standby is required item after August 1, 2017.
8.2. For jumpstart image performance test, you must select “1”, ”2”,”3”, “4”,"5",so add them to “Active Extra Module”by”<<”: Notes:when select "5",you need select related Power port for AC/DC auto switch to “Active Extra Module”
(For example: your machine adapter connect power port 2 of AC/DC Switch,so you need add “[EE Power Switch].PowerPort 2 (JSEE02PT)” to “Active Extra Module”.)
9. Save the task you choosed.
10. Submit start the test task Note:if test result can't follow JS performance spec, Preload will run baseline imge automatically.
The performance measured on the OEM Image must be less than 20% gap from that measured on the Baseline Image
