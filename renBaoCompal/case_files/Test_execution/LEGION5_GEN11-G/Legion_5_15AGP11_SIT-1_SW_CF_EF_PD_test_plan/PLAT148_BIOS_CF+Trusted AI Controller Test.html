Section : Preparation (20) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
Type Confirm

 Note: Don't run the AI APP(like AI now, creator zone etc.) function test together with this TAC tool test case, or the generate/clear key may conflict between APP and tool.



Confirm with the BIOS team that the TAC type is LA1, LA3, LA4 or not supported. For the products with the LA4 chip which need to follow the LA1 test steps and expect the result.  From Gen11, follow the latest SPEC to update LA1 to the highest priority.
BIOS/AI FW installed
2. Install the latest BIOS/AI FW if have for the test system
Tool

3. Copy the TAC tool to the test system before test. Note: This tool need run with admin privileges and run the correct version for X86 and Arm platforms(X86:tactest_x64, Arm:tactest_arm64).
 
Section : TAC Test (30) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
TAC type
1. Run the CMD with admin privileges and negative to the tool folder, run the TAC tool.
It wil return the TAC type of the system support. Please confirm the result expected.

2. Run the TAC tool again, check the tool prompt expected options for LA1 or LA3 a. Tool will prompt user choose one of below 5 actions for LA1 --- 0:Generate Key 1:Get key 2:Get key IdData 3:Clear Key 4: Clear All Key b. Tool will prompt user choose one of below 6 actions for LA3 --- 0:Generate Key 1:Get key 2:Get key IdData 3:Clear Key 4: Clear All Key 5: Enable GenKey

Make sure the actions options shows as correct for LA1 or LA3



The latest tool is compatible with older platform with v1.1 TAC software interface spec. It may show that system support TAC software interface spec v1.1, please update to support v1.2 or later version, then you need hightlight this to BIOS team to update BIOS code
Clear all Key
3. Input number 4 for the action 4 clear all key, clear all key
It will clear all key success.
Get Key

4. Run the TAC tool again and try to get key by input number 1 then random input a number from 0-7 as the key index

For LA3 projects: It will show ITEWinusbTransKey failed , key and user handle mismatch or failed to open the public.bin.


For LA1 proejcts: It will show failed or Failed to open public.bin



The failed to open public.bin depends on your system have run the TAC test before, if you have not run this tool and no public.bin in the folder, after run get the key it will return "ITEWinusbTransKey failed" or "Failed"
Enable GenKey
5. Run the TAC tool then try to generate key by input 0
If the platform TAC type is LA3, It will show failed, need enable GenKey firstly . If the platform TAC is LA1, it will show Success! Return Key_handle 0x80000
6. This step is only for LA3 type: Run the tool again, enther action enable Genkey by input 5 on keyboard.
It will show enable GenKey: Success.
Generate Key
7. This step is only for LA3 type: Run the tool again, try to Generate key by input number 0
TAC LA3 Generate Key: Enagle GenKey: ITEGetCryptoData get data [0x00] [0x80] Success! Return Key_handle 0x8000 save private key save public key
8. Check the private.bin and public bin file
The two files were generated under the tactest tool folder
9. Generate key again by run the TAC tool and select action 0: Generate Key
If the platform is LA3 type, it will show Failed, need enagle Genkey firstly. If the platform is LA1 type, it will show "Success! Return Key_handle 0x8001" save private key save public key
10. This step is only for LA3 type: Run the tool again, enther action enable Genkey by input 5 on keyboard.
It will show enable GenKey: Success.
11. This step is only for LA3 type: Run the tool again, try to Generate key by input number 0
TAC LA3 Generate Key: Enagle GenKey: ITEGetCryptoData get data [0x01] [0x80] Success! Return Key_handle 0x8001 save private key save public key
12. Check the private.bin and public bin
The two files were generated under the tactest tool folder

13.Repeat generate key 6 times, LA3 platform need to enable GenKey then Generate Key

It will generate key and show success. Return key handle from 0x8002 to 0x8007

14. Try to Generate key again

It will show Failed, no free key available as only maximum 8 group keys can be generated.



Note: start from Gen X and later LA3 chip platforms, it support more than 8 group keys to be generated which is not an issue.
Get Key
15. Run the TAC tool again and select the action "1:Get Key:by enter 1

16. Try to Get a key by random enter a key index from 0 to 6

It will show failed or "ITEWinusbTransKey failed. Key and user handle mismatch" as last key generated is 0x8007
Get Key and Key IdData

17.Try to Get key 0x8007 by run tactest again and Enter Key Index 7. For LA3 on Gen X and later platforms,you need to get the key 0x8008 you generate in previous step 14.

It will show "Get Key success".

18. Get Key IdData random enter a number from 0 to 7

Get Key IdData success and print it out.It will all show as [87][87][87]…[87] by default.
Clear Key

19. Run the TAC tool again and try to clear the key by random enter a number from 0 to 7, for example enter key index 3

It will show success.

20. Try to generate key by enter a "0", you may need enable genkey first if the type is LA3.

It will show success, return key handle 0x8003 if cleared index 3 key in previous step and saved private key and public key in the tool folder

21. Try to generate key again.

It will show failed, no free key available. For GenX and later LA3 platform, it still can generate key

22. Try to get key you generate in step 16 by enter key index 3 as example

It will show "Get Key success" and print it out.
Clear all key

23. Try to clear all key by enter 4

It will show success.

24. Try to get key by enter key index 0

It will show failed, key and user handle mismatch
 
Section : Condition Test (30) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
Reboot
1. Generate a Key 0x8000, you may need enable GenKey for LA3 type projects.
2. Try to get the key by enter action 1:Get Key and Enter key index 0
It will show Get Key success.
3. Try to get the Key IdData by enter action 2 and enter key index 0
It will show success and return a group of [87]
4. Reboot system and run the TAC tool again. Repeat the steps 2 and 3
Tool run with no errors and get key and get key IdData all show success
Shutdown
5. Shutdown system, run the TAC tool again and repeat the steps 2 and 3
Tool run with no errors and get key and get key IdData all show success
Force shutdown
6. Force shutdown the system by press the power button for over 15 seconds, run the TAC tool again and repeat the steps 2 and 3
Tool run with no errors and get key and get key IdData all show success
Power Loss
7. Generate a Key 0x8001 by run the tool again and generate another key, you may need enable GenKey for LA3 type projects.
8. Try to get the key by enter action 1:Get Key and Enter key index 1
It will show Get Key success.
9. Try to get the Key IdData by enter action 2 and enter key index 1
It will show success and return a group of [87]
10. Remove both the AC and battery and plug back after 1 minutes. Power on the system and run TAC tool. Repeat the steps 8 and 9
Tool run with no errors and get key and get key IdData all show success
Update BIOS
11. Roll back the BIOS to a previous version and try to generate the key 0x8002 Note: Previous version BIOS may not support the TAC test, confirm with RD and bypass this section if not support
12. Try to get the key by enter action 1:Get Key and Enter key index 2
It will show Get Key success.
13. Try to get the Key IdData by enter action 2 and enter key index 2
It will show success and return a group of [87]
14. Update the BIOS to latest and repeat steps 12 and 13
Tool run with no errors and get key and get key IdData all show success
Update AI FW
15. Roll back the AI FW to a previous version and try to generate the key 0x8003 Note: Previous version AI FW may not support the TAC test, confirm with RD and bypass this section if not support
16. Try to get the key by enter action 1:Get Key and Enter key index 3
It will show Get Key success.
17. Try to get the Key IdData by enter action 2 and enter key index 3
It will show success and return a group of [87]
18. Update the AI FW to latest and repeat steps 16 and 17
Tool run with no errors and get key and get key IdData all show success
