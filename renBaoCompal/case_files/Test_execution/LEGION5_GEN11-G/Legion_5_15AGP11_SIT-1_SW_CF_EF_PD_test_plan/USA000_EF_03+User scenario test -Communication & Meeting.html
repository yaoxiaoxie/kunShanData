APP Installed

Note: This installation step is required before each user scenario case. You can skip this step in other cases if all apps have installed in this case.
Install all below APPs with latest version before scenario testing:

Chrome. Firefox, MSFT Office, Wechat, Teams, Photoshop, Premiere Pro, OneDrive, Adobe Acrobat Reader,VLC Mediaplay, Steam, Wegame

Log Clear

a. Clear the system and application event log from below path: Win+X-> Event Viewer->Windows Log->System log/Application log->Clear log in the actions window.
b. Clear the system reliability history: Search "View reliability history" from the windows menu, it will show all the computer's reliability and problem history, click " view all problems reports" in the bottom button, then click "clear all problem reports".
Log Tool

Open a command prompt window with elevated permission(Run as administrator) and navigate to the directory that contains the ML_Scenario tool, run the "ML_Scenario.bat" to start the latency log capture, and keep it in the background during the test.
IO Device

Please cover different external devices for that test.
① External Keyboard and Mouse
② External Monitor (cover 4K monitor)
③ External Headphone and Speaker
④ External U Disk and Storage
APP Launch Time

In the below test, please check all the APP launch time, you can use the stopwatch in the mobile phone to record this time: Start: Click the shortcuts or press enter to launch End: The APP first UI comes out(not including the complete loading files) 

Make sure the APP launch time will not be over 5 seconds. If >5s, tracking and try again.


If the APP launch time still over 5s, report issue.


Note: 3rd party APP launch time may over 10 or 20 seconds when launch as 1st time, this is expected behavior on Windows 11 as the SAC(Smart APP Control) is on or in evaluation status which will evaluate the security of the APPs.


The 2nd APP launch will not be impacted, so you can ignore the APP launch time in the 1st time.

Meeting and education

Microsoft Effect Pack(MEP) Basic information

1. The graphics driver version should support the MEP feature.
2. The Windows camera app version should be 2022.2210.9.0 or newer.
3. The OS version build should be SV2-22621.1344 or newer.


For Intel platform:


Before testing, please confirm with the project team whether all three drivers are included in the preload.


The test should be conducted after all three drivers are installed:


1) Intel VPU extension driver
2) MSFT MEP driver
3) Camera MEP driver.

Open Settings→Bluetooth & devices→Camera→Integrated Camera

Ensure that all features in Windows Studio effects are visible.

Enable "Portrait Light" and check the preview image in the Settings.


(If MEP is V1 version, please skip testing steps for this feature.) 

Ensure that the preview screen remains almost unchanged in a normal office environment.

Enable "Eye contact" in settings and switch on "Eye contact", then carefully observe the eyes.


(If MEP is V2 version, please switch on "Enhanced.")

Ensure that Eye Contact adjusts the user's line of sight to rise.

Enable "Background effects" in settings and select one of the following options: 1. Standard blur 2. Portrait blur.

Ensure that it has an effect on the preview image.

Enable "Creative filters" in settings and select one of the following options: 1. Illustrated 2. Animated 3. Watercolor.

Ensure that it has an effect on the preview image.

Open Windows camera app to preview, and check NPU usage from "Task Manager"->"Performance".

Ensure NPU usage is higher than zero.
