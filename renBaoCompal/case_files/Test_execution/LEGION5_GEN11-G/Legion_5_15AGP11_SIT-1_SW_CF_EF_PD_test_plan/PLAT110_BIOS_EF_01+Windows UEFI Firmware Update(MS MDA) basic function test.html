Section : Firmware update (15) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :

Notes:


1. Enabled Secure Boot in BIOS setup (for develop BIOS, please skip)


2. For MS project, please login Windows with MSFT account


3. For non-MS project, please enable bitlocker manually with professional OS


4.Turn off BitLocker to test if the test package is not signed by Microsoft

Login system with MSFT account.


Go to settings -> Accounts -> Your info, choose "Sign in with a Microsoft account instead",then input account and password -> Set Facial Recognition and Fingerprint(if support), -> Set PIN.

Plugin AC and charge battery until battery is at least 30% charged.
open CMD , type msinfo32 , to check BIOS version.
Record BIOS version .
Launch Device Manager. This can be done by: Pressing the WIN+X keys then select Device Manager. Or, From the start button select Run then enter: DevMgmt.msc.
In the menu bar, select View, then select Devices by Type (this is typically the default view).
Click Firmware , ensure there must be 1 or more child nodes under the Firmware node.
right click "System Firmware", select Properties and then select the Details tab on the properties dialog.
ensure "System Firmware" exist.
In the Property drop down select Hardware Ids.

The Hardware ID property value must: Be in the format of UEFI\RES_{GUID} where {GUID} is a proper GUID string.


Example: UEFI\RES_{5a2d987b-cb39-42fe-a4cf-d5d0abae3a08}


If so they must all begin with the format of UEFI\RES_{GUID}.


They may have additional components appended to the base component.


For example: UEFI\RES_{5a2d987b-cb39-42fe-a4cf-d5d0abae3a08}&REV_30B01C2
In the Property drop down select Firmware resource version.
Record the value (for example, 030b01c2) for later reference.
Close the dialog window by clicking OK.
Launch Device Manager.
Right-click the System Firmware node and select Update Driver Software… .
In the Update Driver Software dialog, select Browse my computer for driver software.
Select the directory where the UpdateCapsule is located .
The TestCapsule BIOS Package will be installed and a reboot will be required.
TestCapsule BIOS Package update OK .
Manual reboot your unit . After reboot , unit will start flash BIOS.

Three components to the firmware update boot screen , please compare with attach SPEC:


1. OEM logo . For firmware update boot screens the logo must be the same size, position and quality expected as is displayed during the normal boot process


2. The update text . you can see text saying something similar to "please wait while we install a system update" . this may be displayed very briefly so you may miss it.


3. Flash process bar. Ensure the flash bar show normal .

All elems show correct , follow SPEC.

During the flash process :


1. Try to press power button, ensure no action, flash process go on normal .


2. Try to and hold power button for 4 seconds , ensure no action, flash process go on normal .


3. Try to press some keys on keyboard , like : ctrl + alt + del ... , please try 3 types at least. ensure no action, flash process go on normal .


4. Try to hold some keys on keyboard , like : shift , enter .....please try 3 types at least. ensure no action, flash process go on normal .


5. Remove AC


6. Close the lid and then open it

Ensure the BIOS can update OK.


Ensure the system don't enter s3 or s4 or other abnormal behavior.


Ensure the flash process go on normal .


Ensure that when the flash completes, the system enters OS successfully .


There are no errors.
Once Windows is loaded, logon
system logon OK .
Launch Device Manager. This can be done by: Pressing the WIN+X keys then select Device Manager. Or, From the start button select Run then enter: DevMgmt.msc.
In the menu bar, select View, then select Devices by Type (this is typically the default view).
Click Firmware , ensure there must be 1 or more child nodes under the Firmware node.
right click "System Firmware", select Properties and then select the Details tab on the properties dialog.
In the Property drop down select Hardware Ids.

The Hardware ID property value must: Be in the format of UEFI\RES_{GUID} where {GUID} is a proper GUID string.


Example: UEFI\RES_{5a2d987b-cb39-42fe-a4cf-d5d0abae3a08} If so they must all begin with the format of UEFI\RES_{GUID}.


They may have additional components appended to the base component.


For example: UEFI\RES_{5a2d987b-cb39-42fe-a4cf-d5d0abae3a08}&REV_30B01C2

the HWID show OK .
In the Property drop down select Firmware resource version.
Compare the Firmware resource version value with the value you recorded previous. The current value must be different than the recorded value.
If the current version is the same then an error may have occurred during the update process.
Close the dialog window by clicking OK.
Check the “Problem Code” property value.
ensure the value show 0 .
open CMD , type msinfo32 , to check BIOS version.
Compare the BIOS version with your record previous, ensure now the BIOS version is a higher value than what is previous installed on the system.
Go to BIOS setup, check TPM and Secure Boot status.
Ensure TPM and Secure Boot status is enabled.
Load default, save and exit. Boot to OS, check bitlocker status.
System can enter OS successfully, bitlocker is enabled.
