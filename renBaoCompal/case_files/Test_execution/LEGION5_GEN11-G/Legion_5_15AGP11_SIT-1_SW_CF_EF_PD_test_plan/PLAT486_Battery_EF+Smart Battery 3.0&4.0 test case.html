Battery health（only lenovo vantage app support）

Prepare


1.Before testing this function, please confirm with the SWTPM or PPA whether the BIOS/EC supports this function; if not, please skip it


2.Before testing, please seek help from the "project Battery Owner" to provide the tools that can enable/disable temporary mode

1.Launch Vantage-Device Details- Power page- Battery health and check the original UI


SoH >= 80% 

1.Battery Capacity: Battery image is in green ;


Note：If the battery capacity cannot available,it will show grey out,please key issue to battery part.

********Below steps need to flash the Battery FW and check the Vantage UI, need to take some notes*************


1.Tester need to get the "Battery FW package" by different battery manufactor from each project battery owner RD for the “Battery Health” test


2. The "Battery FW package" could change the different status of Battery capacity SOH(FCC/DC, DC means design capacity): SoH>=80%; 40%<= SoH<80%；20%<= SoH<40%; SoH <20%.


3. Before flashing the FW, tester needs to confirm the attention and the risk of the flashing. (like whether could result failure to boot...)

Flash the Battery FW and check the Vantage UI


40%<= SoH<80%

1.Shows "Fair" on the Top-right of the card


2.A battery warning message will be displayed

Flash the Battery FW and check the Vantage UI


20%<=SoH <40%

1.Shows "Poor" on the top-right of the card


2.A battery warning message will be displayed

Note: When updating the battery FW, it is recommended to perform relevant tests at high battery levels. Otherwise, there is a risk of permanent damage to the battery. Please confirm with the battery owner the precautions for updating FW

Flash the Battery FW(tip7;tip 8) and check the Vantage UI


SoH<20%

Section : Battery Night Charging Optimization Mode (80) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
Battery Night Charging Optimiztion Mode

Note: Please confirm with TPM/PPA/SWPM whether your project support smart battery 3.0 feature, if not, pls skip below test. And this section only aim to verify the BIOS/EC/Battery FW if follow the Battery Night Charging Optimization Mode spec design, all function check need the APP spec ready. Prepare test tool first before test.
Check the battery 0x50 Bit0 status from tool
Battery 0X50 Bit0 status is support, which means support Battery Night Charging Mode

Check the battery 0x50 Bit1 status from tool


Note： Check night charge mode defaut is disabled in Vantage or Smart Engine or PCM, If the UI of BNCO is not ready in Vantage or Smart Engine or PCM, you can use tool to check the default settings.

Battery 0X50 Bit1 default status is disabled. which means Night Charging Mode is disabled
Check the Battery Night Charging Mode Enter/Exit time
Enter time (6~11:59PM, 12h format) and exit time (5~10AM,12 format)
When Battery Night Charging Mode disable, insert AC to the unit which RSOC is 85% during night time(bettween the Battery Night Charging Mode Enter/Exit time)
RSOC will increase to 100% continuous at night time, not keep 80% for a period of time.

Set the Battery Night Charging Mode to enabled from Vantage or Smart Engine or PCM.


Note:


1.For support Smart Engine app Project, can open Night Charging Mode via Smart Engine. 联想智能引擎->硬件性能->充电设置


2.If the UI of BNCO is not ready in Lenovo Vantage or Smart Engine or PCM, you can enable/disable it from test tool.

Ensure Battery Night Charging Mode is enabled.

When Battery Night Charging Mode enable, insert AC to the unit which RSOC is 75% at Battery Night Charging Mode enter time without heavy loading, check "Logging" and click "start scan" in test tool

From x90 project, battery icon show ♥ after night charge mode enter time.


Refer to attachment "battery icon show ♥"

Next day, Stop scan after exit time, check time and corresponding RSOC(0x0D).


0x0D is battery RSOC in log.

1. Battery start charging to 80% and stop at 80% (T1 phase);


2. Battery capacity keep at 80% (T2 phase);


3. Battery start re-charging from 80% (T3 phase),


Make sure 100% capacity at Night Charging mode exit time.


Make sure the battery icon is displayed normally without ♥ after night charge mode exit time.


Note:


1. T1/T2/T3 phase may be different for different machine
2. Under T2 phase, the battery capacity is allowed to drop 1% after a longer time of idle due to its own power consumption.

Prepare another machine,When Battery Night Charging Mode enable, insert AC to the unit which RSOC is 85% at Battery Night Charging Mode enter time without heavy loading, check "Logging" and click "start scan"

Ensure battery icon show ♥ after night charge mode enter time.

Next day, Stop scan after exit time, check time and corresponding RSOC(0x0D).


0x0D is battery RSOC in log.

1. Battery stop at 85% (T1 phase);


2. Battery capacity keep at 85% (T2 phase);


3. Battery start re-charging from 85% (T3 phase),


Make sure 100% capacity at Night Charging mode exit time.


Make sure the battery icon is displayed normally without ♥ after night charge mode exit time.


Note:


1. T1/T2/T3 phase may be different for different machine
2. Under T2 phase, the battery capacity is allowed to drop 1% after a longer time of idle due to its own power consumption.

When AC plug in, check battery 0X50 Bit7 status from test tool

Battery 0x50 Bit7 status is AC plug in when AC plug in

When AC plug out, check battery 0X50 Bit7 status from test tool

Battery 0x50 Bit7 status is AC plug out when AC plug out

Change different system time and read battery 0x52 gauge time by tool, and set the system into S3/MS, then resume


 


1. For battery time sync, you can 'scan' again to confirm whether the battery time is synchronized after resume system from MS.

1. Instant system time is written to battery 0x52 (check battery guage time by test tool) when system return to S0;


2. The gap between current system time and battery 0x52 time (0x52 guage time) should <=1 minute.

Change different system time and read battery 0x52 gauge time by tool, and set the system into S4, then resume

1. Instant system time is written to battery 0x52 (check battery guage time by test tool) when system return to S0;


2. The gap between current system time and battery 0x52 time (0x52 guage time) should <=1 minute.

Change different system time and read battery 0x52 gauge time by tool, and set the system into S5, then resume

1. Instant system time is written to battery 0x52 (check battery guage time by test tool) when system return to S0;


2. The gap between current system time and battery 0x52 time (0x52 guage time) should <=1 minute.

Sync up battery time and system time, record the time.


1. Then check the battery time (0x52 gauge time) after 12 hours(keep the sytem in S0)

Battery time gap (0x52 guage time) should <= 15 minutes after 12hours by comparing with system time

2. Then check the battery time (0x52 guage time) after 24 hours(keep the sytem in S0)

Battery time gap (0x52 guage time) should <= 30 minutes after 24hours by comparing with system time
