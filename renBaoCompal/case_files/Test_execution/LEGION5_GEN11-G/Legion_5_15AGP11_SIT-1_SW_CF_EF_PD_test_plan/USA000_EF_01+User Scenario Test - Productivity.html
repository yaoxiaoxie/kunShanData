System Login

Login to a system using Microsoft account. Update system time and connect to WiFi AP.
APP Installed

Note: This installation step is required before each user scenario case. You can skip this step in other cases if all apps have installed in this case.
Install all below APPs with latest version before scenario testing:
EN OS:
Chrome. Firefox, MSFT Office, Wechat, Teams, Photoshop, Premiere Pro, OneDrive, Adobe Acrobat Reader,VLC Mediaplay, Steam, Wegame
CH OS:
MSFT Edge, Lenovo Browser,WPS, Wechat, QQ, QQ Live, IQiyi, TikTok, Tencent Meeting, Teams, Photoshop, Jianying, Steam, Wegame, NetEase Cloud Music, Youdao translate, Baidu Netdisk, 360 Total Security, WinRAR,Sogou Pinyin, Adobe Acrobat Reader
Both EN &CN OS:
For DCC project: Download AutoCAD
For UMA Project: Download & install the LOL via WeGame
For DIS Project:
Non Gaming: Download & install the LOL via WeGame, Download & Install "Genshin Impact" game
Gaming: Download & Install the LOL via WeGame, Download & Install "Genshin Impact" game, Download & Install Cyberpunk 2077 via Steam

All APP installed without any problems and no lag observed
Log Clear

a. Clear the system and application event log from below path: Win+X-> Event Viewer->Windows Log->System log/Application log->Clear log in the actions window



b. Clear the system reliability history: Search "View reliability history" from the windows menu, it will show all the computer's reliability and problem history, click " view all problems reports" in the bottom button, then click "clear all problem reports".
Log Tool

Open a command prompt window with elevated permission(Run as administrator) and navigate to the directory that contains the ML_Scenario tool, run the "ML_Scenario.bat" to start the latency log capture, and keep it in the background during the test.
I/O Device

Please cover different external devices that this case will required
 1. External Keyboard and Mouse include USB, WiFi and BT types
 2. External Monitor 
 3. External Headset


 4. Active Pen for Touchpanel SKU


 
APP Launch Time

In the below test, please check all the APP launch time, you can use the stopwatch in the mobile phone to record this time:


Start: Click the shortcuts or press enter to launch End: The APP first UI comes out(not including the complete loading files)

Make sure the APP launch time will not be over 5 seconds. If >5s, tracking and try again.  If the APP launch time still over 5s, report issue.



Note: 3rd party APP launch time may over 10 or 20 seconds when launch as 1st time, this is expected behavior on Windows 11 as the SAC(Smart APP Control) is on or in evaluation status which will evaluate the security of the APPs. The 2nd APP launch will not be impacted, so you can ignore the APP launch time in the 1st time.

Latency Check

After the test is completed, close the ML_Scenario.exe command window, and the log will auto-save to the tool folder as xxxx.csv, rename it to as this format "project name_test case_date"(For example: Thinkbook 14 G5+ IRH_Productivity_1101.csv), save and upload it to test case result.
Event Log

Save the system and application log by running the system event log tool "EventLogAnalyzer" provided as an attachment. Check the System/APP Event Log

The "EventLogAnalyzer" will auto save the system event log and a "html" file named by "SysEventlog_username_date", open the html file and check if any log with EventID highlighted by red color. If have, submit issue, others EventID were for reference.
Reliability History

Check the reliability history log by search "view reliability history".

Make sure there's no critical events in the reliability monitor. Focus on error related to the Lenovo APP like PCM/Vantage/Baiying etc. Ignore others if only occurs once, highlight the critical event occurs over 3 times.
