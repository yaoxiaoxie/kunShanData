Section : Preparation (10) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
1. Prepare test unit and ensure it is the latest preload image and BIOS/EC/AI FW version.
2. Complete OOBE, then open device manager to check all drivers have no yellow mark
3. Install the test tools: Cinebench R23, Furmark, Intel PTAT, Hwinfo

4. Ask the project leader for the thermal table to ensure the power spec in Balance mode (Legion) or Intelligent mode(Non-Legion)
 
Section : DIS SKU-Compatibility test (180) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
System compatibilty
1.1 Plug in AC and boot into system
1.2 Go to BIOS Setup->Configuration to check Acoustic and Sound sync default status
Ensure this function is disabled
1.3 Enable "Acoustic and Sound sync" function, then save and exit
1.4 Switch the system to Balance mode (Gaming platform) or Intelligent mode(SMB platform)
1.5 Plug the 3.5mm Headset to the unit
1.6 Set the system volume to 30 and then play the audio file in repeated mode Note: keep the audio playing in background, do not close it
1.7 Launch the Furmark to start the GPU stress test Note: keep it running in background always, do not close it
1.8 Launch HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
1.9 Put the system into Sleep for a while, then wake up Note: If the audio pauses, press "Play" to continue playing it
1.10 Launch HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
1.11 Set the system vloume to 50 and wait for 10s
1.12 Launch HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
PM Tests
2.1 Restart the system
2.2 Play the audio file in repeated mode Note:keep it running in background, do not close it
2.3 Launch the Furmark to start the GPU stress test Note: keep it running in background always, do not close it
2.4 Open HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
2.5 Set the system vloume to 100 and wait for 10s
2.6 Open HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
2.7 Shutdown the system and wait for 30s, then boot into the system
2.8 Play the audio file in repeated mode Note:keep it running in background, do not close it
2.9 Launch the Furmark to start the GPU stress test Note: keep it running in background always, do not close it
2.10 Open HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
2.11 Mute the system and wait for 10s
2.12 Open HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is the same as the GPU power Max Limit spec
2.13 Unmute the system and wait for 10s
2.14 Open HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
Hotplug test
3.1 Do the 3.5mm Headset unplug and plug test for 5 cycles
3.2 Open HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
3.3 Unplug and plug the AC adapter for 5 cycles
3.4 Open HWinfo64 to start logging the GPU power for 3min, then stop logging and check the log
Ensure the stable GPU power is higher than the GPU power Max Limit spec
 
Section : UMA SKU-Compatibility test (180) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :

System compatibilty test
1.1 Plug in AC adapter (>=stander)

1.2 Enter the BIOS Setup page->Config->Power, check "Acoustic and Sound sync" default status

Ensure default is disabled

1.3 Enable it, then save and exit.

1.4 Enter OS and switch to Balance mode (Legion) or Intelligent mode(Non-Legion)

1.5 Launch the Intel PTAT with admin, start monitor the "MMIO Power Limit_1 Power" value


Note: keep it running in background, do not close it

1.6 Then launch the Cinebench R23, start the CPU test


Note: keep the cinebench window in foreground and adjust it smaller to monitor the "MMIO Power Limit_1 Power" value

1.7 Set the system volume to 0 and then play the audio file in repeated mode, check "MMIO Power Limit_1 Power" value


Note: minimize audio window to taskbar

Ensure "MMIO Power Limit_1 Power" value meet the APM_PL1 spec

1.8 Turn up the system volume to low level (20~40) and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

1.9 Turn up the system volume to medium level (50~70) and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

1.10 Turn up the system volume to high level (80~100) and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

1.11 Mute the system via hotkey and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value will meet the APM_PL1 spec

1.12 Unmute the system via hotkey and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

Headset Hotplug test (run the Cinebench R23+Audio playback in the whole test)

2.1 Plug the 3.5mm Headset into unit

2.2 Set the volume via hotkey to low level (20~40) and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

2.3 Set the volume via hotkey to medium level (50~70) and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

2.4 Set the volume via hotkey to high level (80~100) and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

2.5 Unplug the headset

2.6 Set the volume via hotkey to low level (20~40) and wait for 10s, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

PM Tests (run the Cinebench R23+Audio playback in the whole test)

3.1 Put the system into sleep mode then resume


Note: if the audio is paused, play it again ; if need, close the PTAT UI and reopen it

3.2 Turn up the system volume to high level, check "MMIO Power Limit_1 Power" value

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

3.3 Restat the system

3.4 Launch the Intel PTAT with admin, start monitor the "MMIO Power Limit_1 Power" value


Note: keep it running in background, do not close it

3.5 Then launch the Cinebench R23, start the CPU test


Note: keep the cinebench window in foreground and adjust it smaller to monitor the "MMIO Power Limit_1 Power" value

3.6 Set the system volume to low level and then play the audio file in repeated mode, check "MMIO Power Limit_1 Power" value


Note: minimize audio window to taskbar

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec

3.7 Shutdown the system then boot again

3.8 Launch the Intel PTAT with admin, start monitor the "MMIO Power Limit_1 Power" value


Note: keep it running in background, do not close it

3.9 Then launch the Cinebench R23, start the CPU test


Note: keep the cinebench window in foreground and adjust it smaller to monitor the "MMIO Power Limit_1 Power" value

3.10 Set the system volume to medium level and then play the audio file in repeated mode, check "MMIO Power Limit_1 Power" value


Note: minimize audio window to taskbar

Ensure "MMIO Power Limit_1 Power" value is higher than the APM_PL1 spec
