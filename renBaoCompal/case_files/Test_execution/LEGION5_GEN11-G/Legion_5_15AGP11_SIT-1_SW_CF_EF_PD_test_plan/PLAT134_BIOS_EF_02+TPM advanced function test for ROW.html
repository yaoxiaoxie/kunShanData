Workloading : (Min) LeadingTime : (Min) Phase :

Notes: this section test only for project support Modern Standby, please cover all TPM modules, if not, please skip this test



For the Chinese system, garbled characters are displayed on the recovery page which is a known problem, details please refer to attachment. Any other types font display issue, please submit issue for further analyze

Go to BIOS setup, enable secure boot, enable Intel Platform Trust Technology(Intel PTT) or Security Chip or AMD Platform Security Processor(AMD PSP) or Pluton, save and exit.

plug in AC, download the latest preload, during OOBE, sign in with Microsoft accounts.

Go to Settings -> Privacy & Security -> Device Encrypted, check default setting.

Ensure Device Encrypted default setting is turned on.
Run command as Administrator: manage-bde -status, check all Volumes' Conversion Status.
System will auto do Device Encrypted action. During encryt, Conversion Status will show "Encryption in Progress", wait for some time until Percentage Encrypted show 100%, Conversion Status will show "Used Space Only Encrypted".
Go to Disk Management, check C disk bitlocker status.
Ensure C disk show BitLocker Encrypted.
Flash the latest BIOS, check bitlocker status.
Ensure system can enter OS successfully and bitlocker is encrypted.
Go to BIOS setup, check secure boot, OA3, TPM setting.
Ensure OA status is kept (if system is actived), secure boot is enabled, TPM is enabled.
Load default in BIOS setup, save and exit.
Ensure system can enter OS successfully.
Shutdown system, then change disk to another machine, then power on system, ensure system prompt to enter bitlocker Recovery key.

Go to


https://account.microsoft.com/devices/recoverykey?refd=support.microsoft.com


website, and login microsoft account.

Ensure website show Recovery key.

Input the wrong recovery key which includes at least four wrong characters

Ensure system cannot enter OS.

Input right Recovery key.

Ensure system can enter OS without any error.


Note: before enter desktop, prompt "Something happened and your PIN isn't available.Click to set up your PIN again",this is design.


Microsoft responded: trust is bult between Windows on the SSD and the TPM in the motherboard of the hardware, so if you exchange the SSD to a new machine it is by design that you would lose device-trust backed features such as bit locker and Windows Hello.

Section : Power Management (60) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :

> Enable TPM


Note: If TPM does not support enable/disable, please skip this section.
if there have option in BIOS setup Goto BIOS setup -> Enable TPM option.
Save and reboot to OS .
Type "TPM.MSC" to enter TPM Management.
Ensure the status in TPM Management show TPM ready.
TPM ready.

Check device manager

For Pluton firmware TPM, the "Microsoft Pluton processor" item appears in Device Manager->Security Device. 


For dTPM, Intel PTT, or AMD PSP, make sure the "Trusted Platform Module 2.0" item appears in the Security device of the device manager.
Manual S3/MS test for 10 times ,When resume : 1. Check all hotkey 2. Check all external port
Ensure all hotkey work normal Ensure all external port work normal
Manual hibernate test for 10 times ,When resume : 1. Check all hotkey 2. Check all external port
Ensure all hotkey work normal Ensure all external port work normal
Manual reboot test for 10 times ,When resume : 1. Check all hotkey 2. Check all external port
Ensure all hotkey work normal Ensure all external port work normal
> Disable TPM
if there have option in BIOS setup Goto BIOS setup -> Disable TPM option.

Save and reboot to OS .

Ensure the system boot to OS normally, no BSOD, hang up and error.
Type "TPM.MSC" to enter TPM Management.
Ensure the status in TPM Management show TPM can't be found
TPM can't be found

Check device manager

For Pluton firmware TPM, Make sure the "Trusted Platform Module 2.0" item disappears in the Security device of the device manager after disabling Pluton 
For dTPM, Intel PTT, or AMD PSP, Make sure the "Trusted Platform Module 2.0" item disappears device manager.

Restart sysrem boot to BIOS setup menu, enable TPM option.

Save and reboot to OS, check device manager -> security devices

Make sure the "Trusted Platform Module 2.0" item appear in Security device of device manager.
