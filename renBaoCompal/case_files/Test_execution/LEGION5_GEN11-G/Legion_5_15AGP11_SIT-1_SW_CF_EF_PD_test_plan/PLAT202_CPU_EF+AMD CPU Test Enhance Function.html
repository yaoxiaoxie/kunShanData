Section : Stress Test (40) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
1. TCT stress
Note: Set ITS/Fn+Q mode to Intelligent Cooling if support. Note: Make sure battery is fully charged to 100% before test.
2. Launch AMD system deck.

3. Start to log all available parameters from AVT. (AVT-> switch to "Log" tab -> click the "Start" button on Log all available parameters)
4. Idle 15 minutes.
5. Click on the "Run" button of TCT widget.
6. Select "Boost Mode Test" under "Test Names"(if CPU support).
7. Setup Switches as 100 for all cores.
8. Uncheck "Disable Thermal Throttling" under "Power and Thermal Configuration".
9. Check "Run continuously" under "Test Druation".
10. Click on "Start" button to run the test.
No errors.
11. Stop TCT test after 8 hours.
Verify no battery drain issue occurred.

12. Stop logging, and check the log file.

1. Verify the values of the Prochot column are all 0(prochot is not triggered).


2. Check CPU power meets the SPEC of the project.


  a. CPU package power should be in boost state(Fast PPT limit & Slow PPT limit) at the beginning of TCT stress.


  b. for the project that does not support STT, CPU package power should be stable at Sustained Power Limit.


  c. for the project that supports STT, CPU package power should be stable at a fixed value(need to confirm with thermal RD) eventually.
13. Repeat step 1 to step 12 on BSM power modes and power supply modes. *For step 5 - step 11, please replace TCT to AIDA64 FPU. (Tools -> check "Stress FPU" -> Start)
Verify CPU clock cannot be down to extremely slow(like 400 MHz) by PROCHOT.
13.1. AC - Best performance: idle 15mins + AIDA64 FPU 15 mins.
13.2. AC - Better Battery: idle 15mins + AIDA64 FPU 15 mins.
13.3. DC - Best performance: idle 15mins + AIDA64 FPU 15 mins.
13.4. DC - Better performance: idle 15mins + AIDA64 FPU 2H(depending on battery capacity).
13.5. DC - Better Battery: idle 15mins + AIDA64 FPU 15 mins.
13.6. DC - Battery Saver: idle 15mins + AIDA64 FPU 15 mins.
