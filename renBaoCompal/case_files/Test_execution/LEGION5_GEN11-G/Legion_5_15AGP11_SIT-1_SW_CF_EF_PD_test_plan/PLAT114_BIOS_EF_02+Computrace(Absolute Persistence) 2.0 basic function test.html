Section : Prepare (10) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :

Only computrace 2.0 is supported in the below steps
Flash the latest BIOS and download the latest preload.
The Serial Number in SMBIOS type 1 needs to be a valid value and must be unique on each device. It cannot be the default value, such as Non/Unique SN or 1234567890.

Check the Platform ID is not 0x00000001 by AbtValidate tool


Note: If the Platform ID is 0x00000001, it is NOT for mass production.
Prepare absolute account, if you don't have valid account, contact biossupport@absolute.com.

The system time must be synced in Windows settings before testing.


System settings ->Time & language ->Date & time -> turn on "Set time zone automatically"

The system shows the current time and time zone

The test device needs to have stable internet connection.


Noteï¼šThere may be network problem via Chinese internet, if so, please change your network.

Section : Download Agent Installer (5) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :

1. Using a web browser, log in to https://cc.absolute.com/ with your Absolute account Username and Password
2. On the navigation bar, click Settings > Agent Management

3. Click 'Get Secure Endpoint Agent'. The 'Get Secure Endpoint Agent <version>' dialog opens
4. Select 'Only download the core agent'.
5. Click Windows.
The zipped agent package is downloaded with the following name: AbsoluteWinCoreAgent-<version>-<account ID>.zip

AbsoluteCoreAgent.msi is one of the extracted files.
7. Launch AbtValiddate,
8. Select Tools > Agent Installers.
The Agent Installers tab opens.
Click Browse, navigate to, and select the AbsoluteCoreAgent.msi file you extracted in step 6.
Click Close.
 
Section : Download and Install API Token (3) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :

1. On the Web Console tab in AbtValidate, click 'New' and enter your username.

2. Click 'Generate'. The Token Manager dialog opens. Leave this dialog open while you complete steps 3 to 4.

3. Using a web browser, log in to https://cc.absolute.com/ with your Absolute account Username and Password.

3.1. On the navigation bar, click Settings > API Management.

3.2. On the API Management page, click 'Create API Token'.

3.3. In Add title, enter your username.



Note:Recommend the same username as in step 1

3.4. In the Token Key section, select ‘Upload public key’.

3.5. Copy the value shown in the Token/Key field on the Token Manager dialog in AbtValidate

3.6. Paste the value in the textbox under Upload public key

3.7.  In the Expiration section, click the expiration date field and select an expiry date. If you don't know what to set the expiration to, set it two days in the future (D+2Days).

3.8. In the Permissions section, leave the permissions and the assigned device group set to their default values

3.9 In the Approved IP Addresses section, enter your public IP if you know it. Otherwise, leave this field blank.

3.10 Click Save.

The Token Key Details section shows the token ID. 

4. Click Copy in the Token ID field and then paste the value in the Token ID field on the Token Manager dialog in AbtValidate.

5. Click OK to save your changes and close the Token Manager dialog.

6. Click X in the Secure Endpoint Console to finish adding the new API token

The Devices in this account field should now show the correct value.

Section : APM Self-Healing test (5) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :
Reboot the test device in BIOS setup, Enable secure boot. Save and exit.
1. Launch AbtValidate.
2. On the Firmware tab, confirm that 'Persistence activated'
Make sure the 'Persistence activated' is selected.
3. On the Dashboard tab, click 'Load Defaults'.
Tabbed interface can be restored to default
4. Select 'Self-Heal' on the Dashboard tab,
5. Click 'Run'.

6. Wait a while and the system will reboot automatically  (Depend on network quality)
7. Wait for test complete, check the result and secure boot status.
The result of the Self-Heal test case is PASS, Secure Boot is enabled.
8. Reboot the test device in BIOS setup, toggle Secure Boot, then repeat steps 1-7.
The result of the Self-Heal test case is PASS, Secure Boot is Disabled.
9. Reboot the test device in BIOS setup, Enable Secure boot and save and exit.

Section : APM "Permanently Disable" Advance function check (10) Attachment :
Workloading : (Min) LeadingTime : (Min) Phase :

1. Reboot system to enter BIOS setup.

2. Check 'Absolute Persistence(R) Module Activation'

Make sure option 'Absolute Persistence(R) Module Activation‘ exists

3. Set Supervisor password, Save and exit


Note: For Consumer products that do not support the WMI function, you can manually change the APM setting to 'Permanently Disable' and the "supervisor password" is show as "administrator password" under BlOS setup.

3.1. Change "Absolute Persistence(R) Module" to Permanently Disabled with SetConfigPassword.vbs tool


Run command as Administrator Powershell:


cscript .\SetConfigPassword.vbs "AbsolutePersistence(R) Module" "Permanently Disabled" svp,ascii,us


svpï¼šCurrent Supervisor password


Note: the WMI names of the options need to refer to the project's LIstBiosSelections log.

"AbsolutePersistence ® Module" change to "Permanently Disabled" success

3.2. Restart system and boot to BIOS setup

3.3. Check 'Absolute Persistence(R) Module Activation

Absolute Persistence(R) Module Activation items will be hidden

3.4 Delete Supervisor password, save and exit

4. UEFI variable flag check


Note: Please contact with project BIOS team to gat check the Permanently Disabled result test tool.

4.1 Run UEFI variable flag check tool and check result.


Note: Refer to attachment "Variable flag sample.png"

Make sure clear status flag is 1

5. Crisis recovery test

5.1 Put crisis BIOS in U disk root directory and attach U disk to the system.

5.2 Shutdown unit , Press Fn+R ,Press power button to power on, system will auto enter the Crisis recovery mode.

Ensure system enter Bios crisis recovery mode normally

5.3 Wait some minutes (5 minutes or more), when flash compeletly, system will auto reboot

Ensure system reboot successfully.

5.4 Reboot to BIOS setup and check APM.

Make sure option 'Absolute Persistence(R) Module' does not exist after crisis recovery.

6. BIOS self healing recovery


Note:If not support self healing function, skip it.

6.1 Copy BiosSelfHealing tool to EFI shell mode U disk and attach the U disk to system.

6.2 Shutdown system than boot to UEFI shell U disk

6.3 Go to BiosSelfHealing folder.Run command: BiosSelfHealing.efi -fvmain in EFI shell mode.

Ensure system will prompt below warning message:


This will destroy the BIOS region and then auto reboot, which may cause unit failto bootup !


Please make sure your BIOS can support the BIOS self-healing function!!!


Press "Enter" to continue, or "q" to quit

6.4 Press "Enter" key.

System maybe reboot for three times. This progress may take a long time, please wait.

6.5 Check prompt message after recovery successfully.

System has self-healed by restoring BIOS from backup.


Please check BIOS version and then update BIOS if needed.


Press Esc key to hide this message.

6.6 Press "Esc" key or wait for 5 seconds.

System will reboot normally

6.7 Reboot to BIOS setup and check APM.

Make sure option 'Absolute Persistence(R) Module' does not exist after BIOS self healing recovery.
