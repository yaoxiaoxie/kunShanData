Section : Offload stream playback Attachment :
Workloading : 100(Min) LeadingTime : 100(Min) Phase : Test Log Mandatory : N

1. Open *.wav/.MP3 file with Media player
N
2. Click on Pause while stream is playing
Stream should pause
N
3. Click on play
Stream should continue to play and there should not be any glitches. Note: Progress bar will stop when it is Paused.
N
4. While stream is playing change the player volume
Volume change should effect to the stream
N
5. Mute/Unmute the player when stream is active
Silent data when Player is Muted. Audio should come when Player is Unmuted and there are no glitches Note: Progress bar of music player is still moving when it is muted
N
6. Perform System Mute/Unmute when stream is active
Silent data when system is Muted. audio should resume when system is Unmuted and there are no glitches Note: Progress bar of music player is still moving when system is muted
N
7. While stream is playing change the system volume
Volume change should effect to the stream and there are no glitches
N
8. Change the resolution ,right click on the desktop-->Display Settings-->Resolution
No effect should be there in Audio playback when Screen Resolution changes
N
9. Restore to default resolution
N

Method1 Time out: Control panel->hardware and sound->power options->change when the computer sleeps->set ‘turn off the display’ time
N

Method2 Close lid: Control panel->hardware and sound->power options->choose what closing the lid does->set ‘when I close lid’ to do nothing
N
10. While playing turn off the display if support.
Audio should continue without any delay
N
11. Turn on display
Audio should continue without any delay
N
12. While streams are playing, seek to forward and backward
There should not be any glitches when doing seek. Playback should be smooth
N
13. Enable repeat option for the Player
While repeating there should not be any glitches and repeat should be successful
N
14. Perform Seek backward and forward again
No glitches when seek is performed
N
15. Perform Seek till the EOF
No glitches when seek is performed
N
16. Close and open the players when streams are active and running
Verify the sound, it should be audible with no glitches
N
17. During playback,plug out/in AC several times.
Audio should continue without glitches
N
18. Put the system to Sleep(MS/S3)(Please cover lid close and start menu) & Hibernate & WarmBoot & ColdBoot and resume, playback audio
Audio can playback without glitches
N
19. Repeat the setp 18 , 5 times
Audio can playback without glitches
N
20. Play the music again , Please cover different audio formats,as below, Channels: mono, 2ch, 3ch, 4ch, 6ch, 8ch. Frequency: 8k to 192 kHz. Bit depth: 8, 16 & 24
Verify the sound, it should be audible with no glitches Note: Audio effects such as Channel Converter will be used if the stream format is different with endpoint format. Ex: 5.1Ch ,8 bit, 44.1K stream is played on the 2ch,16bit, 48K endpoint default format.
N

Section : Sine/Sweep test with spectrogram for Offload Attachment :
Workloading : 5(Min) LeadingTime : 5(Min) Phase : Test Log Mandatory : N
1. Disable Microphone Effect
N
2. Open Audacity->Edit->Preferences->Devices-> Host: Windows WASAPI, Playback Device:internal Speaker, Recording Device:internal speaker->Click OK
N
3. Open Audacity->Edit->Preference->Tracks->Spectrograms->Set Max Frequency to 20000 and set Window size to 1024.
N
4. In Audacity, start Recording, then playback 2Ch_8bit_8k_Sweep with Groove Music, stop recording.
N
5. Click on "Audio Track"(Left side of Audacity window) then select "Spectrogram", check the spectrogram of the recording audio
No distortion should be observed in Audacity spectrogram Notes: If observed spectrogram is a diagonal line, no distortion. Tester can also compare the spectrograms of 2Ch_8bit_8k_Sweep.wav and the recording audio to check
N
